## FAQ Quick Reference

<AccordionGroup>
  <Accordion title="How do I configure API keys?">
    Read [Configuration and Environment](/en/detailed_guide/config). Pay special attention to the provider-to-model-list structure in `provider.json`.
  </Accordion>
  <Accordion title="Do the decorators support synchronous functions?">
    No. `@llm_function`, `@llm_chat`, and `@tool` all require `async def` functions.
  </Accordion>
  <Accordion title="How do I build multi-turn chat?">
    Start with [llm_chat Decorator](/en/detailed_guide/llm_chat) and learn how `history`, `stream=True`, and return modes work.
  </Accordion>
  <Accordion title="How do I interrupt the current response?">
    Read [Abort and Cancellation](/en/detailed_guide/abort) and use `AbortSignal`.
  </Accordion>
  <Accordion title="How do I let the model call functions or external APIs?">
    Read [Tool System](/en/detailed_guide/tool) to learn `@tool`, return types, and tool guidance injection.
  </Accordion>
  <Accordion title="Which LLM providers are supported?">
    Read [LLM Interface Layer](/en/detailed_guide/llm_interface). The framework is built around OpenAI-compatible interfaces and can connect to many compatible services.
  </Accordion>
  <Accordion title="How should I debug failures or retry issues?">
    Start with the troubleshooting section in [LLM Interface Layer](/en/detailed_guide/llm_interface), then use logs and the event stream for deeper inspection.
  </Accordion>
</AccordionGroup>
